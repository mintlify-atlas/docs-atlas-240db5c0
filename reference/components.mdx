## NetworkFlux

The main page component. Manages application state and renders the input form alongside the React Flow canvas that displays the network graph.

This component accepts no props. It holds the following internal state:

| State variable | Type | Description |
|---|---|---|
| `jsonData` | `string` | Raw adjacency matrix input from the user, as a JSON string. |
| `maxFlowExpected` | `string` | The expected max flow value entered by the user. Compared against the computed `maxFlow` to determine whether the sink node displays the lamp-on or lamp-off image. |
| `stepStop` | `number \| undefined` | When set, instructs the algorithm to stop after a specific augmentation step, enabling step-by-step visualization. |
| `verifyMaxFluxStarted` | `boolean` | Tracks whether the max flow computation has been initiated, used to control rendering of the graph. |
| `maxFlowInitialFlux` | `number \| undefined` | Stores the max flow value from a previous computation pass to preserve it across re-renders or step-based replays. |

## TextUpdaterNode

A custom React Flow node component that renders each vertex in the network graph. Its appearance changes depending on whether it represents the source, sink, or an intermediate node.

### Visual behavior

- **Source node** (`node === 0`): Displays a server/stack image (`stack.jpg`).
- **Sink node** (`node === nodesQuantity - 1`): Displays a lamp image. Shows `lamp-on.jpeg` when `maxFlow >= maxFlowExpected`; shows `lamp-off.jpeg` otherwise.
- **Intermediate nodes**: Display the node's numeric index as a text label centered within the circle.

### Handle positioning

Target handles (incoming edges) default to `Position.Left`. For intermediate-to-intermediate connections, the position is adjusted based on the parity of the node indices:
- Odd node receiving from an even node → `Position.Top`
- Even node receiving from an odd node → `Position.Bottom`

Source handles (outgoing edges) follow the same parity logic in reverse, defaulting to `Position.Right`.

Handles to/from the source node (index `0`) or sink node (index `nodesQuantity - 1`) always use the default Left/Right positions.

## CustomEdge

A custom React Flow edge component that renders directed edges between nodes. It automatically detects bidirectional connections and curves them to prevent overlap.

### Bidirectional detection

`CustomEdge` subscribes to the React Flow store via `useStore` and checks whether a reverse edge exists between the same pair of nodes (i.e., an edge where `source === target` of the current edge and `target === source`). When a reverse edge is detected, it uses `getSpecialPath` instead of the standard bezier path to visually separate the two edges.

The offset passed to `getSpecialPath` is `+30` when `sourceX < targetX` (left-to-right edge) and `-30` when `sourceX >= targetX` (right-to-left edge).

## getSpecialPath

A helper function that generates a quadratic bezier SVG path string with a perpendicular offset. Used by `CustomEdge` to visually separate bidirectional edges that would otherwise overlap.

```typescript
getSpecialPath(params: GetSpecialPathParams, offset: number): string
```

### Parameters

<ParamField path="params" type="GetSpecialPathParams" required>
  Coordinates of the edge endpoints.

  <Expandable title="GetSpecialPathParams properties">
    <ParamField path="params.sourceX" type="number" required>
      X coordinate of the edge source.
    </ParamField>
    <ParamField path="params.sourceY" type="number" required>
      Y coordinate of the edge source.
    </ParamField>
    <ParamField path="params.targetX" type="number" required>
      X coordinate of the edge target.
    </ParamField>
    <ParamField path="params.targetY" type="number" required>
      Y coordinate of the edge target.
    </ParamField>
  </Expandable>
</ParamField>

<ParamField path="offset" type="number" required>
  Displacement applied to the bezier control point. Positive values curve the path down and to the right; negative values curve it up and to the left. `CustomEdge` passes `+30` or `-30` depending on edge direction.
</ParamField>

### Return value

Returns an SVG path string using a quadratic bezier curve:

```
M {sourceX} {sourceY} Q {centerX + offset} {centerY + offset} {targetX} {targetY}
```

Where `centerX = (sourceX + targetX) / 2` and `centerY = (sourceY + targetY) / 2`. The control point is shifted by `offset` in both the X and Y directions, producing a diagonal curve that separates the two edges of a bidirectional pair.
