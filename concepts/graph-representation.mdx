title: Graph representation

NetworkFlux & MaxFlux Generator accepts graphs as **JSON adjacency matrices** — a format that is compact, human-readable, and maps directly to the internal data structures used by the algorithm.

## Adjacency matrix format

A graph with `n` nodes is represented as an `n × n` two-dimensional array:

```json
[
  [row 0],
  [row 1],
  ...
  [row n-1]
]
```

Each element `matrix[i][j]` is the **capacity** of the directed edge from node `i` to node `j`. A value of `0` means no edge exists between those two nodes.

<Info>
The matrix must be square. Every row must have exactly `n` elements, where `n` is also the number of rows.
</Info>

## Node indexing convention

| Index | Role | Label in UI |
|---|---|---|
| `0` | Source | `s` |
| `1` through `n-2` | Intermediate nodes | their numeric index |
| `n-1` (last) | Sink | `t` |

This convention is enforced by the implementation. The `fordFulkerson` call in `verifyMaxFlux.ts` always passes `s = 0` and `t = jsonData.length - 1`:

```typescript verifyMaxFlux.ts
export function verifyMaxFlux(data, stepStop?) {
  let jsonData = data;
  try {
    jsonData = JSON.parse(data);
  } catch (e) { /* empty */ }
  return fordFulkerson(
    jsonData,
    0,
    jsonData.length - 1,
    jsonData.length,
    stepStop
  );
}
```

## A concrete 4-node example

This matrix defines a graph with 4 nodes — source, two intermediate nodes, and sink:

```json
[
  [0, 10, 10,  0],
  [0,  0,  5, 15],
  [0,  0,  0, 10],
  [0,  0,  0,  0]
]
```

Reading every non-zero value:

| `matrix[i][j]` | Meaning |
|---|---|
| `matrix[0][1] = 10` | Edge from `s` to node 1 with capacity 10 |
| `matrix[0][2] = 10` | Edge from `s` to node 2 with capacity 10 |
| `matrix[1][2] = 5`  | Edge from node 1 to node 2 with capacity 5 |
| `matrix[1][3] = 15` | Edge from node 1 to `t` with capacity 15 |
| `matrix[2][3] = 10` | Edge from node 2 to `t` with capacity 10 |

The diagonal and all other cells are `0` because no node has a self-loop and there are no other connections.

## Input constraints

<AccordionGroup>
  <Accordion title="Square matrix">
    The number of rows must equal the number of columns. A matrix with 4 rows must have 4 elements in every row.
  </Accordion>
  <Accordion title="Non-negative integer capacities">
    Every value must be a non-negative integer. Negative capacities and fractional values are not supported by the algorithm.
  </Accordion>
  <Accordion title="At least one path from s to t">
    There must be at least one directed path from index 0 to the last index. If no such path exists, the algorithm returns a maximum flow of 0 immediately.
  </Accordion>
  <Accordion title="Minimum size">
    The matrix must have at least 2 nodes (a 2×2 matrix with just source and sink). Graphs with only 1 node have no edges and no meaningful flow.
  </Accordion>
</AccordionGroup>

## How `generateNodesAndEdges` reads the matrix

The `generateNodesAndEdges` function in `generateNodesAndEdges.ts` iterates over the adjacency matrix to create React Flow node and edge objects:

```typescript generateNodesAndEdges.ts
export default function generateNodesAndEdges(
  data,
  maxFlowExpected,
  maxFlowInitialFlux?
): ReturnType {
  const nodes: Node<any, string>[] = [];
  const edges: Edge<any>[] = [];

  const verifyMaxFluxResponse = verifyMaxFlux(data, null);
  const maxFlow = maxFlowInitialFlux
    ? maxFlowInitialFlux
    : verifyMaxFluxResponse.maxFlow;

  data.forEach((node, nodeIndex) => {
    nodes.push({
      id: nodeIndex.toString(),
      data: {
        label:
          nodeIndex === 0
            ? 's'
            : nodeIndex === data.length - 1
            ? 't'
            : nodeIndex.toString(),
      },
      // position, type, style ...
    });
    // edges are pushed for each positive adjacency value
  });

  return { nodes, edges, graphWidth, maxFlow };
}
```

For each `nodeIndex`, the label is assigned by the same convention: index `0` becomes `'s'`, the last index becomes `'t'`, and every other index becomes its number as a string.

## Visual layout logic

Node positions are computed from the node index and the parity of intermediate nodes:

```typescript generateNodesAndEdges.ts
position: {
  x:
    (countOf2 === 1 && nodeIndex !== data.length - 1
      ? nodeHorizontalSteps - 1
      : nodeHorizontalSteps) * 200,
  y:
    nodeIndex === 0 || nodeIndex === data.length - 1
      ? 125
      : nodeIndex % 2 === 0
      ? 0
      : 250,
},
```

The layout rules are:

- **Source and sink** are placed at `y = 125` — vertically centered.
- **Intermediate nodes with an even index** are placed at `y = 0` — top row.
- **Intermediate nodes with an odd index** are placed at `y = 250` — bottom row.
- Horizontal spacing is `200px` per step.

This produces a layered diagram where intermediate nodes alternate above and below a center line, keeping the source and sink visually anchored at the left and right edges.

<Tip>
For graphs with many intermediate nodes, keep the matrix small enough that the alternating layout remains readable. Graphs beyond 8–10 nodes may produce overlapping edges in the current layout engine.
</Tip>
